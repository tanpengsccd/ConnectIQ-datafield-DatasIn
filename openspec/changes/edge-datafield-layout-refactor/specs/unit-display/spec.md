## 新增需求

### 需求:单位显示

系统必须在数据值的下方显示对应的单位，单位字体必须比数据值字体小一级。

#### 场景:速度单位显示
- **当** 显示速度数据（Speed）
- **那么** 单位 "km/h" 必须显示在数值下方，使用 `FONT_TINY` 字体

#### 场景:功率单位显示
- **当** 显示功率数据（Power）
- **那么** 单位 "W" 必须显示在数值下方，使用 `FONT_TINY` 字体

#### 场景:心率单位显示
- **当** 显示心率数据（HeartRate）
- **那么** 单位 "bpm" 必须显示在数值下方，使用 `FONT_TINY` 字体

### 需求:数据格式规范

系统必须对不同类型的数据值应用统一的格式规则。

#### 场景:速度格式
- **当** 速度值四舍五入取整，不超过 2 位数（最大 99）
- **那么** 速度显示为整数，如 "25"，单位显示为 "km/h"

#### 场景:功率格式
- **当** 功率值取整数，最大 9999
- **那么** 功率显示为整数，如 "180"，单位显示为 "W"

#### 场景:坡度格式
- **当** 坡度值保留小数点后一位
- **那么** 坡度显示为小数，如 "3.5"，单位显示为 "%"

#### 场景:心率格式
- **当** 心率值取整，最大 999
- **那么** 心率显示为整数，如 "145"，单位显示为 "bpm"
