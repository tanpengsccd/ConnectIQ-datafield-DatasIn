## 新增需求

### 需求:字段数量自适应布局

系统必须根据当前显示的字段数量（1/2/3/4），自动选择对应的布局算法进行绘制。

#### 场景:单字段居中显示
- **当** 只显示 1 个字段
- **那么** 数据必须水平垂直居中显示，使用大号字体

#### 场景:双字段上下布局
- **当** 显示 2 个字段
- **那么** 上面字段显示在数据框上半部分，下面字段显示在下半部分，上下顺序由用户在设置中配置

#### 场景:三字段品字布局
- **当** 显示 3 个字段
- **那么** 支持两种品字布局：倒品字（上2小下1大）或正品字（上1大下2小），由用户在设置中配置

#### 场景:四字段田字布局
- **当** 显示 4 个字段
- **那么** 字段以 2×2 网格均匀分布，每个字段占宽高 50%

### 需求:布局尺寸约束

系统必须将数据绘制限制在数据框可用区域内，禁止内容溢出可见边界。

#### 场景:文字边界裁剪
- **当** 文字内容超出数据框宽度
- **那么** 超出的部分必须被裁剪，不显示在数据框外

### 需求:大屏小屏自适应

系统必须根据 `dc.getHeight()` 的返回值判断设备类型，并自动选择合适的字体大小。

#### 场景:小屏设备字体
- **当** `dc.getHeight()` ≤ 200
- **那么** 必须使用 `FONT_TINY` 作为标签字体，`FONT_MEDIUM` 作为数值字体

#### 场景:大屏设备字体
- **当** `dc.getHeight()` > 200
- **那么** 必须使用 `FONT_MEDIUM` 作为标签字体，`FONT_LARGE` 作为数值字体
