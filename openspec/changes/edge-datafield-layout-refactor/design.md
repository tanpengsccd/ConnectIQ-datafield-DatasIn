## 上下文

当前 `datasin1View.mc` 使用 XML 布局文件定义固定位置的 label，通过 `getObscurityFlags()` 处理手表多场景遮挡。对于仅面向 Edge 码表的场景，这些复杂性不必要。

**现有问题：**
- XML label 无边界约束，数据框缩小时文字溢出
- 手表遮挡逻辑冗余
- 固定坐标无法自适应大小屏

## 目标 / 非目标

**目标：**
- 支持 Edge 码表小屏（520/530/540）和大屏（1030/1040/1050/150）
- 数据框高度固定为屏幕 1/5，宽度支持 1/2 或 100%
- 根据字段数量（1/2/3/4）动态选择布局
- 数据值下方显示小字体单位

**非目标：**
- 手表设备支持（移除手表遮挡代码）
- 5 个以上字段支持
- 运行时切换布局配置（编译时固定）

## 决策

### 1. 用代码绘制替代 XML 固定布局

**决策**：在 `onUpdate()` 中使用 `dc.drawText()` 动态计算位置

**理由**：
- Edge 数据框尺寸规则（1/5 高度，1/2 或 100% 宽度），比例计算简单
- 避免 XML 无边界约束导致的溢出问题
- 不同字段数量需要不同布局算法，代码实现更灵活

**替代方案考虑**：
- XML + `locText`：仍需手工设置 width/height，无法真正响应式
- 多种 XML layout + 切换：增加复杂度，效果不如代码计算

### 2. 字段数量布局规则

| 字段数 | 布局 |
|--------|------|
| 1 | 居中大字体 |
| 2 | 上大下小或上小下大（由配置决定） |
| 3 | 倒/正 品字：上 2 小，下 1 大 或者 下 2 小，上 1 大 |
| 4 | 田字：2×2 均匀分布 |

### 3. 大小屏判断

```monkey
var height = dc.getHeight();
var isLargeScreen = height > 200;  // 大屏 > 200px
```

- 小屏：使用 `FONT_TINY` / `FONT_XTINY`
- 大屏：使用 `FONT_MEDIUM` / `FONT_LARGE`

### 4. 单位显示位置

单位文本位于数据值正下方，字体比数据值小一级：
- 数据值：`FONT_LARGE` 或 `FONT_MEDIUM`
- 单位：`FONT_TINY`

## 风险 / 权衡

- **风险**：代码绘制性能略低于 XML 静态布局
  - **缓解**：Connect IQ 设备刷新率低（1Hz），性能影响可忽略
- **权衡**：移除了手表兼容性
  - 用户当前仅面向 Edge，如以后需要手表，需重新添加遮挡逻辑
