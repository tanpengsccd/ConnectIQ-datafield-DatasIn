## 为什么

当前数据字段实现使用 XML 固定布局，存在以下问题：

1. **手表遮挡模式复杂**：通过 `getObscurityFlags()` 处理多种遮挡情况，但实际主要面向 Edge 码表设备，不需要手表场景
2. **文字溢出**：当数据框缩小时（如只占 1/10 屏幕），文字溢出可见区域
3. **无法自适应**：固定坐标在大小屏设备上表现不一致

## 变更内容

1. **移除手表遮挡模式**：删除 `obscurityFlags` 相关逻辑，简化布局系统
2. **简化设备支持**：只支持 Edge 码表（小屏：520/530/540；大屏：1030/1040/1050/150）
3. **固定数据框比例**：只支持高度 1/5 屏幕，宽度 1/2 或 100% 两种规格
4. **按字段数量动态布局**：
   - 1 字段：居中大字体
   - 2 字段：上大下小或上小下大（可配置）
   - 3 字段：倒/正品字排列（上大下小或上小下大,可配置）
   - 4 字段：田字排列
5. **添加单位显示**：数据值下方显示小字体单位

## 功能 (Capabilities)

### 新增功能

- `adaptive-layout`: 根据数据字段数量动态选择布局算法，支持 1/2/3/4 字段场景
- `edge-device-support`: 基于 `dc.getWidth()/getHeight()` 实现大屏小屏自适应
- `unit-display`: 数据值下方显示小字体单位（kph, W, bpm）

### 修改功能

- （无）

## 影响

- **代码**：`source/datasin1View.mc` 需重构 `onLayout` 和 `onUpdate` 逻辑
- **布局**：`resources/layouts/layouts.xml` 中的多种 layout 简化为代码绘制
- **兼容**：移除手表场景代码，但当前代码体积更小、维护更简单
